# Architecture Bootstrap Checklist

An Architect may start work only after answering the following questions:

## Project State

- What is the current phase?
- What sprint is active?
- What milestone is active?
- What RFC is active?

## Architecture

- What is the project's north star?
- What is the source of truth?
- What principles govern the project?
- What architectural boundaries are already established?

## Governance

- What decisions are open?
- What work is in the Implementation Queue?
- What is blocked?
- What is explicitly out of scope?

## Risk

- What architectural risks exist?
- What documentation is missing or outdated?
- What hidden assumptions should be surfaced?

## Readiness

If any answer is unknown, continue reading project documents before proposing changes.

If documents conflict, report the conflict before proposing changes.
