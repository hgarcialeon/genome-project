# Genome Bootstrap Protocol

This protocol is used by any architect, engineer, or automated system joining the Genome project.

Do not rely on chat history.

The repository is the source of truth.

## Bootstrap Steps

Read, in order:

1. `PROJECT_STATE.md`
2. `docs/VISION.md`
3. `docs/CONSTITUTION.md`
4. `docs/ARCHITECT.md`
5. `docs/GOVERNANCE.md`
6. `docs/ARCHITECTURE_BOOTSTRAP_CHECKLIST.md`
7. active RFC from `PROJECT_STATE.md`
8. relevant SPEC files
9. relevant ADRs
10. `IMPLEMENTATION_QUEUE.md`

## Required Output

After bootstrap, produce a Bootstrap Report with:

```markdown
# Bootstrap Report

## Repository Status

## Current Phase

## Active Sprint

## Active RFC

## Current North Star

## Governance State

## Architectural Risks

## Explicitly Out of Scope

## Recommended Objective

## Questions Before Proceeding

## Ready
```

## Rule

Do not propose implementation before producing the Bootstrap Report.
